# GTM Venture Advisors — Landing Page Build Instructions

## Project Overview

Build a single-page landing site for **GTM Venture Advisors**, the capital markets and financial advisory arm of the Toronto Business Development Centre (TBDC). This is a wholly owned subsidiary of TBDC — a government-backed international startup accelerator headquartered in Toronto.

GTM is an **advisory firm** (NOT an investment fund). They help high-growth international founders with capital raising strategy, M&A, IPO/RTO readiness, and go-to-market execution. They do not deploy capital directly or take equity positions. This distinction matters — the copy must never imply they are a VC fund or make direct investments.

**Target audience (in priority order):**
1. TBDC founders being introduced to GTM through accelerator programming
2. External founders looking for capital markets advisory
3. Institutional partners (VCs, PE firms, family offices) evaluating GTM as a co-advisory partner

**NOT targeting:** Co-investors. The page should not read like a fund pitch deck.

## Design System

### Brand Alignment

GTM Venture Advisors is a sub-brand of TBDC. The design should feel like a sibling of the TBDC website — recognizably part of the same family — but with a more finance-forward, institutional tone. Think: the private banking floor of a building whose lobby is TBDC.

--navy-light:  #111D2E    (card backgrounds, subtle elevation)

### Design Patterns (matching TBDC site)

1. **Eyebrow labels**: Small-caps teal text above section headings (e.g., "WHAT WE DO", "OUR TEAM"). Use `text-teal uppercase tracking-[0.15em] text-xs font-medium`.
2. **Gold accent text**: Use for emphasis words in major headlines. Apply the gold color to 1-3 key words, not entire sentences.
3. **Dark hero**: Full-width navy background with subtle gradient or atmospheric overlay. The TBDC site uses a city skyline photo — GTM should use an abstract atmospheric treatment (gradient mesh, subtle noise texture, or geometric pattern) since we don't have photography yet.
4. **Card treatments**: Cards on dark backgrounds get `bg-navy-light border border-navy-mid` with a teal top-border accent on hover. Cards on light backgrounds get `bg-white border border-gray-200 shadow-sm` with hover elevation.
5. **Section rhythm**: Alternate between dark (navy) and light (surface) section backgrounds. Each section gets generous vertical padding (py-24 to py-32).
6. **Horizontal rules**: Use gold accent lines (1px, 60px wide) as section dividers or above headings.
7. **CTA buttons**: Primary = `bg-teal text-white hover:bg-teal-light`. Secondary = `border border-white/30 text-white hover:border-teal hover:text-teal`.

### Animations (Framer Motion)

- **Scroll-triggered fade-up**: Every section's content should animate in on scroll. Use `whileInView` with `initial={{ opacity: 0, y: 30 }}` and `animate={{ opacity: 1, y: 0 }}` with `transition={{ duration: 0.6 }}`.
- **Staggered children**: Card grids and service lists should stagger with `transition={{ delay: index * 0.1 }}`.
- **Hero entrance**: Title and subtitle animate in sequentially on page load (not scroll). Title first (0.3s delay), subtitle (0.6s), CTA (0.9s).
- **Parallax on hero**: Subtle parallax on background elements (5-10% scroll rate difference).
- **Hover micro-interactions**: Cards lift slightly on hover (`whileHover={{ y: -4 }}`). CTAs have subtle scale (`whileHover={{ scale: 1.02 }}`).
- **Counter animations**: Stat numbers (80+ years, $1B+, 3 sectors) should count up when scrolled into view.

### Layout

- **Max content width**: 1200px, centered
- **Section padding**: `py-24 md:py-32`
- **Grid**: Services/cards use `grid grid-cols-1 md:grid-cols-2 lg:grid-cols-3 gap-6`
- **Mobile-first**: All sections must work well on mobile. Cards stack single-column. Hero text scales down. Navigation becomes a hamburger menu.

## Important Implementation Notes

1. **All content is in the app.jsx file.** Content is separated into a data object at the top of the file for easy editing. Do not fetch content from any API or external source.

2. **Smooth scrolling**: Implement smooth scroll-to-anchor behavior for all nav links.

3. **Responsive**: Mobile-first. Test at 375px, 768px, and 1280px breakpoints.

4. **Performance**: Lazy-load below-fold sections. Use `will-change: transform` sparingly.

5. **Accessibility**: All images need alt text. Contrast ratios must meet WCAG AA. Focus states on interactive elements. Semantic HTML (nav, main, section, footer).

6. **No placeholder images**: Use gradient/initial-based placeholders for headshots. Use abstract shapes, not stock photos.

7. **Headshot placeholders**: Create elegant circular placeholders with the person's initials on a subtle gradient background (navy to teal). These will be replaced with real photos later.

8. **The name "GTM Venture Advisors" is a working name** and may change. Structure the code so the name appears in the data object and can be changed in one place.

9. **External links**: The TBDC link (tbdc.com) and email CTA should open in new tabs / default mail client respectively.

10. **Scroll progress indicator**: Add a thin teal progress bar at the very top of the viewport that fills as the user scrolls down the page.
